## 완료한 것 — 자세히

### 데이터 (Phase 0, 4단계 + 검증)

`pipeline/` 안 5개 스크립트:

1. `01_build_characters.py` — 어문회 배정한자 5,978자 xls (`../hanjahanja/docs/reference/한국어문회 배정한자(5978자) 정리.xls`) 파싱 → `characters` 테이블. 음·훈·부수·부수획수·총획·급수(코드+라벨)·장음.
2. `02_enrich_unihan.py` — Unicode Unihan zip 다운로드(캐시 `pipeline/cache/Unihan.zip`) → 5,978자 매칭. 보강 결과:
   - `pinyin` (kMandarin) 5,969자
   - `pinyin_plain` (성조 제거 사본) 5,969자
   - `pinyin_extra` (kHanyuPinyin 다음 발음 JSON) 5,634자
   - `simplified` (kSimplifiedVariant) 2,032자
   - `traditional` (kTraditionalVariant) 139자
   - `en_def` (kDefinition) 5,858자
3. `03_parse_nikl.py` — 표준국어대사전 stdict XML 89개 파일 streaming 파싱(iterparse). 한자 표기는 `original_language_info/original_language` (language_type='한자')에서. 결과:
   - 단어 436,144 / 의미 508,675 / 한자→단어 역인덱스 892,617
   - 한자어 236,182 + 혼종어 88,904 + 고유어 75,750 + 외래어 23,986
4. `04_build_fts.py` — FTS5 가상 테이블 두 개. `unicode61 remove_diacritics 2`로 성조 무관 매칭. **참고: 스키마 변경 시 반드시 04를 다시 돌려야 fts 가상 테이블 재생성됨** (DROP/CREATE 자동 처리).
5. `build_all.py` — 1~4 순차 + `src-tauri/resources/hanjadict.db` 자동 셋업 (mac/linux 심볼릭 링크, win 복사).
6. `verify.py` — 종합 검증 (한자 1자, 한자 → 단어, 한국어 단어, FTS, 동음이의, DB 통계).

DB 최종: 264.9 MB.

### 앱 (Phase 1 + 2)

스택: Tauri 2 + React 19 + Vite 6 + TypeScript 5.8 / Rust + rusqlite (bundled SQLite + FTS5).

레이아웃: 좌측 320px 검색바 + 결과 리스트 / 우측 상세 카드.

검색 라우팅 (`src-tauri/src/db.rs::search`):
- 한자 들어있음 → `characters` 직접 lookup + `words.hanja` 정확 매칭 + FTS 보완
- 한글 → `characters.reading` 정확 매칭 + `words.word_clean` 정확/prefix
- Latin (병음·영문) → `fts_chars` (pinyin, pinyin_plain, en_def 모두 검색)

UI 컴포넌트:
- `SearchBar` — 디바운스 80ms
- `ResultList` — 단어/한자 섹션 분리, 동음 번호 표시
- `CharCard` — 큰 한자(144px) + 음·훈·뱃지 + 중국어(병음/간체/번체) + 영문 + 사용 단어 리스트
- `WordCard` — 큰 한자(56px) + 한자 분해 chips (각 한자 클릭 → 검색) + 뜻풀이

API 클라이언트(`src/lib/api.ts`): Tauri 환경 감지 (`__TAURI_INTERNALS__`) → 실제 invoke / 브라우저 dev → mock 폴백. `pnpm dev`(mock) 와 `pnpm tauri:dev`(실데이터) 두 모드 모두 동작.

### 폰트 (시안 직후 추가)

문제: 시스템 fallback chain이 한자별로 다른 폰트 매핑 → 글자마다 모양 다름.

해결: `@fontsource/noto-serif-{kr,sc,tc}` woff2 region별 임베드.
- KR (어문회 한자, default): 949KB
- SC (간체 칸 `[data-region="sc"]`): 1.4MB
- TC (번체 칸 `[data-region="tc"]`): 1.3MB
- 합 ~3.6MB, 일관된 region 디자인 (예: 国 의 玉 위 점 위치 등 region별 정확)

JP 생략 (한자 사전에서 거의 사용 안 됨, v2에서 필요시 추가).

## 의사결정 / 합의

- **앱 형태**: Tauri 2 데스크탑 앱 (jieum과 동일 스택). 메뉴바·맥 .dictionary 번들·Electron은 모두 검토 후 보류.
- **검색 범위**: 한자 단자 + 한국어 단어 (NIKL stdict). krdict/opendict는 v1.x. 부수+획수 검색은 v2.
- **TTS**: 만다린 발음 v2로 미룸 (시안 단계에서 합의).
- **jieum 통합**: 포기. hanjadict는 사전, jieum은 입력기 — 각자 본분.
- **폰트**: 옵션 B (4-region NotoSerifCJK). 사이즈 차이 미미해서 subset(C) 안 함.
- **저장소**: public, MIT (코드) + 데이터는 각 출처 라이선스. 학생들이 포크/사용 가능.
- **Windows 빌드**: GitHub Actions 자동화 (다음 세션). 진우형 직접 빌드 안 해도 됨.

## 발견한 함정 / 버그 (다시 안 밟도록)

1. **stdict 한자 표기 위치** — 처음에 `<origin>` 찾았는데 거기는 어원(고유어의 ＜용가＞ 같은). 한자어의 한자 표기는 `<original_language_info>/<original_language>` 경로 (`language_type` 값이 '한자', '한자(중)', '한자(일)' 등으로 시작).

2. **stdict word 정규화** — `^`(복합어 구분), `·`(가운뎃점), `-`(접두/접미 표시) 모두 제거 후 `word_clean`으로 검색용 정규화. 동음 번호(`가01` 의 `01`)는 끝 1~2자리 숫자 정규식으로 분리.

3. **FTS5 hanja 컬럼 UNINDEXED 실수** — 처음에 `fts_words.hanja UNINDEXED`로 두면 한자 단어 직접 검색이 안 됨. indexed로 변경.

4. **NFC 병음 ASCII 판정 버그** (release 빌드 후에야 발견) — `db.rs`의 `is_ascii_only` 분기가 "guó"의 'ó'(U+00F3 NFC)를 ASCII로 못 보고 chars 분기에서 빠짐 → 병음 검색 0건. en_def 매칭으로 영문(`edge`)만 됐던 것. **수정**: `is_latin_or_ws`로 Latin-1 Supplement (0x00C0-0x024F) + Latin Extended Additional + combining diacritics 영역 모두 포함.

5. **성조 무관 검색** — DB pinyin은 "guó", 사용자가 보통 "guo"로 입력. `pinyin_plain` 컬럼 추가 + FTS5 `remove_diacritics 2` 토크나이저로 양쪽 정규화. 양 형태 모두 매칭.

6. **Tauri bundle.resources `../` prefix** (release SIGABRT 원인) — `["../resources/hanjadict.db"]`로 두면 `.app/Contents/Resources/_up_/resources/hanjadict.db`로 매핑되는데 `resolve_resource("resources/...")`가 자동으로 `_up_` 매칭 안 함 → setup hook panic → release(`panic="abort"`) SIGABRT. **수정**: `src-tauri/resources/hanjadict.db` 심볼릭 링크 + `tauri.conf.json` `"resources": ["resources/hanjadict.db"]` 단순화. 코드의 `db_path()`는 robust하게 4개 후보 모두 시도(`resources/`, `_up_/resources/`, dev cwd 2개).

7. **dev/prod DB 경로 분리** — dev에서 cwd는 `src-tauri/`, prod .app은 `/`. dev fallback에 `cwd.join("../resources/...")` + `cwd.join("resources/...")` 둘 다 두면 어느 위치에서 실행해도 동작.

8. **글꼴 fallback chain** — `font-family`에 region 미명시 시 한자별 다른 폰트로 매핑. 명시 region별 분리 임베드 (`@fontsource/noto-serif-{kr,sc,tc}`) + `data-region` 속성 또는 클래스로 분기.

9. **release `panic = "abort"`** — Cargo.toml 설정. 어떤 panic도 즉시 SIGABRT, stderr가 .app에서 안 보임. 디버깅 시 macOS Console.app 또는 터미널에서 `/Applications/한자사전.app/Contents/MacOS/hanjadict` 직접 실행하면 stderr 표시. 향후 setup hook에 명시적 에러 dialog 추가하면 학생 디버깅 편함.

10. **Bash CWD 가끔 reset** — 도구로 `cd src-tauri` 후 다음 호출에서 cwd가 hanjadict로 돌아간 경우가 있었음. 절대 경로 사용이 안전.
